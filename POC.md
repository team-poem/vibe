# PoC: audio-capture

V.I.B.E의 첫 번째 PoC. 내장 MacBook 마이크에서 저지연으로 오디오를 받아오는 게
실제로 가능한지 검증한다. 박수 감지 알고리즘이나 Tauri 통합은 다음 PoC에서.

## 목표

- `cpal`로 기본 입력 장치를 열고 콜백으로 샘플을 받아 RMS를 측정한다.
- 입력을 wav로 덤프해서 다음 PoC(`poc/clap-detector`)의 입력 자료로 쓴다.
- 콜백 지연, 샘플 드롭, CPU 사용률을 측정한다.

## 인터페이스 계약

본 프로젝트로 가져갈 때 `capture` 모듈은 아래 콜백 시그니처를 유지한다.

```rust
fn on_samples(samples: &[f32], sample_rate: u32)
```

- `samples`는 mono로 다운믹스된 f32 PCM (-1.0 ~ 1.0)
- 다음 PoC의 박수 감지기는 이 시그니처에 그대로 붙는다.

## 실행

```bash
# 콘솔에 RMS 막대만 출력
cargo run --release

# wav 덤프까지 (인자로 경로 지정)
cargo run --release -- samples/test.wav
```

처음 실행 시 macOS가 마이크 권한 다이얼로그를 띄운다. 허용해야 캡처가 동작한다.
권한이 거부됐거나 입력 장치가 없으면 즉시 에러로 종료된다 (조용히 실패시키지 않음).

## 출력 해석

- `[device]` 라인: 잡힌 장치 이름, 샘플레이트, 채널, 포맷.
- `[rms] peak ... dBFS`: 200ms 윈도우의 피크 RMS를 dBFS로 표시. 박수 치면 -10dBFS
  근처까지 솟고, 정적은 -50dBFS 아래로 가라앉아야 정상.
- `[stats]` 라인: 5초마다 콜백 횟수, 누적 샘플 수, 콜백 사이 최대 간격(ms).
  최대 간격이 20ms 이상으로 자주 튀면 백그라운드 부하나 장치 이슈를 의심.

## 측정 결과

MacBook Air 내장 마이크 / 약 5분 캡처.

- 장치: MacBook Air 마이크 / 48 kHz / 1 ch / F32
- 콜백 max_gap: 11.25 ~ 11.55 ms (목표 ≤ 20 ms 만족)
- 캡처 안정성: 드롭 없이 연속 콜백 (5분간 27,676 콜백)
- 노이즈 플로어 (EMA 추정): ~ -65 dBFS
- 박수 피크: -1.8 / -3.8 dBFS  (+63~65 dB above floor)
- 타이핑 피크: -52 ~ -57 dBFS  (+15~20 dB above floor)
- 캡처 품질 (afplay 검증): 박수·타이핑·말소리 또렷하게 들림
- 모은 wav 샘플:
  - [x] 통합 (박수 + 타이핑 + 말소리, `samples/test.wav`)
  - [ ] 박수 단독 / 이중 (선택)
  - [ ] 타이핑 단독 (선택)
  - [ ] 기침 (선택)
  - [ ] 음악 (선택)
  - [ ] 말소리 단독 (선택)
  - [ ] 문 닫힘 (선택)
